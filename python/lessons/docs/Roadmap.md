# Python Mastery — Full 6-Month Roadmap

---

## PHASE 1 — Core Language Foundations (Weeks 1–4)

| Week | Topics |
|------|--------|
| 1 | How Python works, syntax, values, types, variables, memory model |
| 2 | Strings, numbers, booleans — deep behavior, not just usage |
| 3 | Control flow — `if/else`, `for`, `while`, how they execute internally |
| 4 | Functions — definition, arguments, return values, scope |

---

## PHASE 2 — Deep Dive into Functions (Weeks 5–8)

| Week | Topics |
|------|--------|
| 5 | Call stack, execution frames, how Python runs your code |
| 6 | Recursion — mechanics, stack depth, when to use it |
| 7 | Closures, lexical scoping, `nonlocal` |
| 8 | Higher-order functions, `*args`, `**kwargs`, pure vs impure functions |

---

## PHASE 3 — Functional Programming (Weeks 9–12)

| Week | Topics |
|------|--------|
| 9 | Immutability, why it matters, mutable vs immutable types in CPython |
| 10 | `map`, `filter`, `reduce` — internals and alternatives |
| 11 | Function composition, currying with `functools.partial` |
| 12 | Generators, `yield`, lazy evaluation, iterators from scratch |

---

## PHASE 4 — Object-Oriented Programming (Weeks 13–17)

| Week | Topics |
|------|--------|
| 13 | Objects in memory — `id()`, `__dict__`, how instances are stored |
| 14 | Classes — `__init__`, methods, `self`, class vs instance attributes |
| 15 | Inheritance, MRO (Method Resolution Order), `super()` |
| 16 | Encapsulation, `@property`, name mangling, dunder methods |
| 17 | OOP vs FP trade-offs — when to use which |

---

## PHASE 5 — Modules, Libraries & Architecture (Weeks 18–21)

| Week | Topics |
|------|--------|
| 18 | Import system — how `import` works internally, `sys.modules`, `PYTHONPATH` |
| 19 | Packages, `__init__.py`, namespaces, relative imports |
| 20 | Standard library deep dive — `itertools`, `functools`, `collections`, `pathlib` |
| 21 | Virtual environments, `pip`, dependency management, writing reusable modules |

---

## PHASE 6 — Advanced Internals (Weeks 22–26)

| Week | Topics |
|------|--------|
| 22 | CPython execution model — bytecode, `dis` module, the interpreter loop |
| 23 | Memory — stack vs heap, reference counting, garbage collection |
| 24 | Concurrency — threads, GIL, `asyncio`, event loop internals |
| 25 | Decorators, metaclasses, descriptors |
| 26 | Performance — profiling, `__slots__`, avoiding common bottlenecks |
